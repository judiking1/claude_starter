# Retro Skill — 주간 회고

Git 히스토리를 분석하여 엔지니어링 회고를 자동 생성합니다.

## 사용법
- `/retro` — 최근 7일 회고 (기본)
- `/retro 24h` — 최근 24시간
- `/retro 14d` — 최근 14일
- `/retro 30d` — 최근 30일

## 분석 단계

### Step 1: Raw Data 수집
```bash
git log --oneline --since="<period>" --format="%H|%ai|%an|%s"
git diff --stat HEAD~N
```

### Step 2: 핵심 지표 계산
| 지표 | 설명 |
|------|------|
| 총 커밋 수 | 기간 내 main 커밋 |
| LOC 변화 | insertions / deletions / net |
| 테스트 비율 | test 파일 LOC / 전체 LOC |
| 활동 일수 | 커밋이 있는 날 수 |

### Step 3: 시간대별 분포
24시간 히스토그램으로 작업 시간 패턴 시각화:
```
00-06: ░░
06-09: ░░░░░
09-12: ████████████
12-14: ░░░░
14-18: ██████████
18-21: ░░░░░░
21-24: ░░░
```

### Step 4: 커밋 분류
Conventional Commits prefix 기준 분포:
- `feat`: 새 기능
- `fix`: 버그 수정
- `refactor`: 리팩토링
- `test`: 테스트
- `chore`: 설정/의존성
- `docs`: 문서
- `perf`: 성능 최적화
- `style`: 코드 스타일

파이 차트 형태의 비율 표시.

### Step 5: 핫스팟 분석
가장 자주 변경된 파일 Top 10:
```
12회  src/components/Dashboard.tsx
 8회  src/services/api.ts
 6회  src/hooks/useAuth.ts
```
**높은 churn은 불안정성 신호** — 리팩토링 대상일 수 있음.

### Step 6: 포커스 점수
단일 디렉토리 집중도 계산 (0-100):
- 90+: 깊은 집중 (한 영역에 몰두)
- 60-89: 적정 분산
- 60 미만: 산발적 (컨텍스트 스위칭 과다)

### Step 7: 연속 출근일 (Streak)
연속으로 커밋이 있는 날 수 추적:
- 현재 streak: X일
- 최장 streak: Y일
- 주중 평균 활동일: Z일/주

### Step 8: 이전 회고 대비 (14일+ 기간만)
이전 동일 기간과 비교:
```
커밋 수:  45 → 52  (↑ 15%)
LOC:     +2,100 → +3,400  (↑ 62%)
테스트:  18% → 24%  (↑ 6%p)
Focus:   72 → 85  (↑ 개선)
```

## 출력 형식

```
# 주간 회고 — 2026-03-25 ~ 2026-04-01

## Summary
| 지표 | 값 |
|------|------|
| 커밋 | 52 |
| LOC (net) | +3,400 |
| 테스트 비율 | 24% |
| 활동일 | 5/7 |
| 포커스 점수 | 85/100 |
| 연속 출근일 | 12일 |

## 시간대별 분포
[히스토그램]

## 커밋 분류
feat: 40% | fix: 25% | refactor: 15% | test: 10% | chore: 10%

## 핫스팟
[Top 10 파일]

## 인사이트
- [관찰 1: 특이한 패턴이나 개선점]
- [관찰 2: 잘된 점]
- [제안: 다음 주에 시도할 것]
```

## 톤
- 직접적이고 구체적 (파일명, 숫자로 근거)
- 잘한 점은 구체적으로 칭찬
- 개선점은 투자 관점으로 프레이밍 ("테스트 비율 투자가 장기적으로...")
- 판단은 근거 기반, 감정적 표현 지양

## 한국어로 소통
모든 회고 내용은 한국어. 파일명과 코드 참조는 영어 그대로.
